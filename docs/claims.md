# Guarantees you can break: the claims list

> Generated by `pnpm claims:render`; do not edit by hand. 23 guarantees ("falsifiable claims") across 8 of the 13 parts of the system.

**A "falsifiable claim" is a promise that comes with its own off switch.** Each row is something the
system guarantees, plus the exact bug that would break it and one command to watch a real test catch
that bug. Don't trust the green check, flip the switch and see the test fail. The live pass or fail
verdict is not committed here (it would read stale the moment the code moves); get it from
`pnpm claims:verify` or a CI run.

| Claim | Boundary | Breaks when | Falsify |
|---|---|---|---|
| A webhook signature binds the timestamp, so a captured (body, signature) pair cannot be replayed. | `delivery-edge` | `CHAOS_WEBHOOK_SIGN_BODY_ONLY` | `pnpm prove webhook-signing --break` |
| Every webhook delivery reaches a terminal state; a failed send is retried, never silently dropped. | `delivery-edge` | `CHAOS_WEBHOOK_DROP_ON_FAIL` | `pnpm prove webhook-at-least-once --break` |
| The moderator escalates to a human on a low-confidence verdict instead of guessing (FR5 calibration). | `external-dep` | `MODERATOR_DISABLE_ABSTAIN` | `pnpm prove moderator-abstain --break` |
| The moderator never confidently approves content the precedent flags: an approve that diverges from majority-remove precedent escalates to a human instead. The safety invariant, not gold-set agreement, is the bar — which is what lets the cheaper gpt-5-nano draft model stay safe (a confident-but-wrong approve is caught structurally, not shipped). | `external-dep` | `MODERATOR_DISABLE_APPROVE_GUARD` | `pnpm prove moderator-no-confident-approve-of-flag --break` |
| The moderator fences attacker-controlled post bodies as DATA before they reach the model; disabling the guard leaves them in instruction context. | `external-dep` | `MODERATOR_DISABLE_INPUT_GUARD` | `pnpm prove input-guard-fences-untrusted-body --break` |
| The moderator fences attacker-reachable retrieved context (poisoned precedents / policy text) as DATA before it reaches the model; disabling the corpus guard leaves a stored injection in instruction context. | `external-dep` | `MODERATOR_DISABLE_CORPUS_GUARD` | `pnpm prove retrieved-context-fenced --break` |
| A stored vote is exactly +1 or -1, so a post score can only ever equal (upvotes − downvotes); an out-of-range value can neither enter the votes table nor inflate the score. The ±1 rule lives in one place (VOTE_VALUES) and the request schema, DB CHECK, OpenAPI, and property generator all derive from it. | `process-rest` | `CONTENT_BUG_VOTE_OUT_OF_RANGE` | `pnpm prove vote-value-in-band --break` |
| A stored vote is a member of the set {+1, -1}, not merely within the range [-1, +1]: an in-range but out-of-set value (0) is rejected by the same set-membership DB CHECK that rejects a ±7. The falsifier is the set projection of VOTE_VALUES, the one that catches the adversary a range bound would wave through. | `process-rest` | `CONTENT_BUG_VOTE_OUT_OF_SET` | `pnpm prove vote-value-in-set --break` |
| A community's feed contains exactly its own posts and never another tenant's, even under an arbitrary interleave of cross-community writes (Commitment 9). | `tenancy` | `CONTENT_BUG_TENANT_LEAK` | `pnpm prove tenant-isolation --break` |
| Postgres Row-Level Security is a second tenancy layer beneath the service-layer WHERE: with the service filter removed entirely (SELECT … WHERE true, a deliberately broken service layer), the database still returns only the bound community's rows — RLS catches a service bug the service layer would have leaked. The policy is fail-open when no community is bound, so it can only ever hide a foreign row, never invent a zero-rows failure mode. | `tenancy` | `CONTENT_BUG_DISABLE_RLS` | `pnpm prove rls-blocks-broken-service-layer --break` |
| Every span the deployed system emits carries tenant.id (Commitment 9); a dropped stamp is caught by the live Jaeger audit. | `observability` | `CHAOS_TENANT_SPAN_DROP` | `pnpm prove tenant-span-everywhere --break` |
| The transactional outbox keeps mutating HTTP latency independent of the broker: publishing on the request path breaches the vote SLO even on a healthy broker. | `process-async` | `CHAOS_SYNC_PUBLISH` | `pnpm prove outbox-isolates-broker-latency --break` |
| The events polling read enforces community membership: an authenticated non-member is refused (403 not-a-member), so the REST fallback cannot leak another tenant's event stream — the same isolation the WS upgrade enforces at the edge (ADR-0025). | `tenancy` | `GATEWAY_BUG_SKIP_EVENTS_AUTHZ` | `pnpm prove events-polling-membership --break` |
| The activity feed merges the WebSocket push and the polling fallback by per-community `seq`, so an envelope delivered on both transports renders once — never a duplicate React key or doubled row. | `websocket` | `WEB_BUG_WS_NO_DEDUP` | `pnpm prove ws-merge-dedup --break` |
| An agent that hand-edits the generated OpenAPI (or drifts the Zod schema) and leaves the committed spec behind is caught: the regenerated document no longer equals the committed openapi.yaml, so the Zod round-trip spec — and the `pnpm openapi:verify` drift gate it twins — go red. | `agentic` | `AGENT_DESYNC_OPENAPI` | `pnpm prove agent-cannot-silently-desync --break` |
| An assertion-less agent-authored test has no teeth and is caught by mutation: when the oracle stops asserting (the always-pass / `__eq__`→True attack), the mutated target survives and the mutation gate reds. The in-process twin of the Stryker harness (ADR-0031), falsifiable in milliseconds. | `agentic` | `AGENT_EMIT_ASSERTIONLESS_TEST` | `pnpm prove agent-test-has-teeth --break` |
| A strong invariant gate cannot be gamed: the tenant-isolation property still reds on a leak an agent patched in, even when a weak-oracle agent test stays green around it — oracle strength, not a green check, is what defends the boundary. | `agentic` | `AGENT_PATCH_AROUND_GATE` | `pnpm prove gate-survives-agent-gaming --break` |
| An agent's qaroom-mcp tool-use trajectory is reverse-conformance-checked against an allowed graph — discovery before a tool is invoked by name, the discipline the read-first MCP surface actually permits — and every transition carries agent.id / session.id. A tool call fired outside that graph (a read before the catalogue was discovered) is caught by the reverse-conformance gate, the in-process TypeScript twin of the moderator trajectory-DST (T21). | `agentic` | `AGENT_OFF_GRAPH_TOOL_CALL` | `pnpm prove agent-trajectory-on-model --break` |
| A derived projection cannot silently drift from its single source: the vote-value property generator emits exactly the {+1, -1} set recomputed from VOTE_VALUES, so weakening the deriver while leaving the CODEOWNED invariant pristine — the cheapest chain tamper — is caught by a recompute-and-diff conformance gate. | `agentic` | `AGENT_WEAKEN_VOTE_DERIVER` | `pnpm prove deriver-conformance --break` |
| No span the system emits carries PII (an email-shaped value or a denied body/identifier key); a deliberate leak is caught by the in-process PII-in-spans audit. Commitment 9 stamps tenant.id onto every span — this pins that nothing personal rides along with it. | `observability` | `CHAOS_SPAN_PII` | `pnpm prove pii-free-spans --break` |
| A durable JetStream consumer keeps its lag within the consumer-lag SLO (CONSUMER_LAG_SLO): stall it and num_pending plus the oldest-unacked age climb past the bound, so the backpressure gate reds. The alert threshold and this gate derive from the SAME single source, so a stalled moderator under a burst becomes a defined, caught failure mode instead of a silent one. | `process-async` | `CHAOS_CONSUMER_STALL` | `pnpm prove consumer-lag-bounded --break` |
| A GDPR erasure removes the user from every service: identity deletes its user data and emits a per-community `user.erased` event; content and donations consume it and delete their slice (dedup-guarded, so a redelivery never double-effects). After the cross-service saga settles, no service returns the user. Disabling one service’s erasure handler (CONTENT_BUG_SKIP_ERASURE) leaves that service still returning the user — the saga reaches Incomplete and this property reds. The guarantee is distributed correctness across the cascade, not a single-service delete. | `process-async` | `CONTENT_BUG_SKIP_ERASURE` | `pnpm prove user-erased-everywhere --break` |
| The promotion ledger's verdict logic cannot launder a real red into a flake to advance green_head: a deterministic, reproducible failure classifies as `red`, never `flaky`. Relabelling a confirmed red as flaky — the cheapest Goodhart move on the ledger — is caught by the meta-gate that measures the measure, the same governance the invariant sources sit under (CODEOWNERS + the promotion-ledger-guard diff-over-commits flag). | `agentic` | `LEDGER_RELABEL_RED_AS_FLAKY` | `pnpm prove relabeled-red-stays-red --break` |
